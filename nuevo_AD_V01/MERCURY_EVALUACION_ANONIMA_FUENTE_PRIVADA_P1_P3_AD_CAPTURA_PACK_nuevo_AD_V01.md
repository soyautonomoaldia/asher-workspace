## Area Revisada

Evaluacion anonima P1-P3 para una posible prueba privada controlada de `AD Captura Pack`, usando solo categorias no identificables y una fuente privada externa autorizada por Salva.

## Veredicto

evaluacion anonima condicionada

## Fuente Privada Externa

Estado:

> aprobada por Salva, condicionada

Lectura Mercury:

Salva autoriza usar una fuente privada externa al workspace para identificar candidatos P1-P3, pero Mercury no debe ver, escribir ni conservar nombres, emails, telefonos, empresas, ubicaciones, clientes ni datos identificables en `nuevo_AD_V01`.

La fuente privada externa no queda descrita en este archivo. La evaluacion solo es valida si la fuente existe fuera del workspace y entrega a Mercury categorias anonimas suficientes para P1, P2 y P3. Si esa fuente no existe o no puede operar sin datos identificables, la evaluacion pasa a bloqueada para prueba real.

## Evaluacion P1-P3

No marco ningun perfil como `apto` pleno porque Mercury no ha verificado personas concretas ni disponibilidad. La evaluacion es de categorias anonimas, no de destinatarios reales.

## Categorias Evaluadas No Identificables

Categorias permitidas para cada ID anonimo:

- oficio aproximado;
- decide por si mismo: si / no / dudoso;
- alternativa actual: WhatsApp / notas / libreta / memoria / calendario;
- puede opinar con ejemplos ficticios: si / no / dudoso;
- perdida probable: vuelta / material / cobro / presupuesto simple / cierre / ninguna clara;
- riesgo de complacencia: bajo / medio / alto;
- riesgo de pedir datos reales: bajo / medio / alto;
- invalidadores probables: factura / cobro / pago / foto / calendario real / WhatsApp real / normativa / certificacion / integracion / backend / ninguno evidente.

No se evaluan ni registran nombres, emails, telefonos, empresas, ubicaciones, clientes, datos de trabajos, facturas, fotos, audios, chats, importes, calendarios, cobros, pagos ni documentos.

## Riesgos De Sesgo Y Complacencia

- P1 puede dar feedback amable si procede de un referido cercano; la evidencia solo cuenta si compara espontaneamente contra WhatsApp/notas y nombra una perdida concreta.
- P2 puede parecer buen contraste por oficio, pero contaminar la prueba si deriva a certificaciones, normativa, seguridad tecnica, factura o fotos.
- P3 puede ampliar demasiado el scope si representa reparacion general compleja o mezcla varios oficios; debe mantenerse en trabajos cortos y decisiones simples.
- La seleccion desde fuente privada puede sobrerrepresentar perfiles cercanos a Salva. Esto no invalida la conversacion, pero debilita cualquier lectura de demanda o canal.

## Riesgos De Datos Reales

Riesgos a vigilar:

- que el candidato quiera explicar un caso real;
- que mencione nombres, ubicaciones, clientes, importes o documentos;
- que necesite fotos reales, facturas, tickets, chats, calendario o WhatsApp real para opinar;
- que P2 necesite detalles tecnicos, normativa, boletines o certificaciones;
- que la conversacion derive a precio, producto disponible, demo operativa, backend o integracion.

Si aparece cualquiera de estos riesgos como condicion para valorar, debe activarse parada o bloqueo. No debe convertirse en backlog.

## Invalidadores

Invalidadores duros:

- no existe fuente privada externa real fuera del workspace;
- la fuente exige registrar identificadores en `nuevo_AD_V01`;
- el candidato no puede opinar con ejemplos ficticios;
- el candidato necesita datos reales para valorar;
- el valor depende de WhatsApp real, calendario real, foto, factura, cobro, pago, backend o integracion;
- P2 deriva a normativa, certificacion, alcance industrial o seguridad tecnica;
- no hay alternativa actual clara;
- no aparece perdida concreta frente a WhatsApp, notas, libreta, memoria o calendario;
- el contacto o envio se intenta ejecutar sin gate final separado de Salva.

## Recomendacion Final

Mantener la evaluacion anonima como condicionada y usar P1-P3 solo como IDs anonimos de trabajo.

Recomendacion operativa:

> Continuar solo si la fuente privada externa entrega categorias anonimas suficientes para P1, P2 y P3 sin exponer identificadores a Mercury ni escribir datos personales en `nuevo_AD_V01`. Si no puede hacerlo, bloquear la prueba real y volver a ensayo interno ficticio.

Antes de cualquier contacto sigue haciendo falta aprobacion final separada de Salva sobre destinatarios concretos gestionados fuera del workspace, texto final, owner de contacto, owner de sesion, consentimiento, ventana de envio, registro anonimo, almacenamiento, borrado, invalidadores y criterio de parada.
